# Peeple Jobs - 외국인 잡 포털 개발 계획서

## 1. 프로젝트 개요

### 1.1 서비스 소개
**Peeple Jobs**는 한국에서 일하고자 하는 외국인 구직자와 기업을 연결하는 전문 잡 포털 플랫폼입니다.

| 항목 | 내용 |
|------|------|
| **프로젝트명** | Peeple Jobs |
| **목표** | 외국인 구직자 맞춤형 취업 플랫폼 |
| **프레임워크** | Next.js 14+ (App Router) |
| **데이터베이스** | Supabase (PostgreSQL) |
| **인증** | Supabase Auth |
| **배포** | Vercel |

### 1.2 핵심 기능
- 직업 정보 제공 (업종별/지역별 맞춤형 공고)
- 다국어 지원 (한국어, 영어, 중국어 등)
- 이력서/자기소개서 작성 지원
- 헤드헌팅 서비스
- 명함 관리 솔루션

### 1.3 유사 서비스 분석
**Giggle** (외국인 유학생 아르바이트 플랫폼)
- 간편한 공고 등록
- 맞춤형 공고 검색
- 시간제 취업허가 서류 작성 지원
- 다국어 지원 (영어/한국어)

## 2. 사용자 유형 및 역할

| 사용자 | 주요 기능 |
|--------|----------|
| **Job Seeker (구직자)** | 회원가입/로그인, 이력서 관리, 공고 검색/지원, 커뮤니티 |

## 5. 데이터베이스 설계

### 5.1 주요 테이블

| 테이블 | 설명 |
|--------|------|
| `users` | 구직자 사용자 정보 |
| `companies` | 기업 정보 |
| `resumes` | 이력서 정보 |
| `job_postings` | 채용 공고 |
| `applications` | 지원 내역 |
| `bookmarks` | 스크랩/관심 공고 |
| `subscriptions` | 기업 구독 정보 |
| `chat_rooms` | 채팅방 |
| `chat_messages` | 채팅 메시지 |
| `notifications` | 알림 |
| `reports` | 신고 내역 |

### 5.2 핵심 테이블 스키마

#### users
```sql
CREATE TABLE users (
    id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    email VARCHAR(255) UNIQUE,
    phone VARCHAR(20),
    name VARCHAR(100) NOT NULL,
    profile_image_url TEXT,
    visa_type VARCHAR(50),
    nationality VARCHAR(50),
    preferred_language VARCHAR(10) DEFAULT 'ko',
    address JSONB,
    created_at TIMESTAMP WITH TIME ZONE DEFAULT NOW(),
    updated_at TIMESTAMP WITH TIME ZONE DEFAULT NOW(),
    is_deleted BOOLEAN DEFAULT FALSE
);
```

#### companies
```sql
CREATE TABLE companies (
    id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    business_number VARCHAR(20) UNIQUE NOT NULL,
    company_name VARCHAR(200) NOT NULL,
    logo_url TEXT,
    description TEXT,
    benefits JSONB,
    status VARCHAR(20) DEFAULT 'pending',
    created_at TIMESTAMP WITH TIME ZONE DEFAULT NOW(),
    updated_at TIMESTAMP WITH TIME ZONE DEFAULT NOW(),
    is_deleted BOOLEAN DEFAULT FALSE
);
```

#### job_postings
```sql
CREATE TABLE job_postings (
    id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    company_id UUID REFERENCES companies(id),
    title VARCHAR(200) NOT NULL,
    description_ko TEXT,
    description_en TEXT,
    job_type VARCHAR(50),
    location VARCHAR(200),
    salary_range JSONB,
    visa_requirements JSONB,
    status VARCHAR(20) DEFAULT 'draft',
    deadline TIMESTAMP WITH TIME ZONE,
    created_at TIMESTAMP WITH TIME ZONE DEFAULT NOW(),
    updated_at TIMESTAMP WITH TIME ZONE DEFAULT NOW(),
    is_deleted BOOLEAN DEFAULT FALSE
);
```

## 8. API 설계

### 8.1 RESTful API 엔드포인트

#### Auth
```
POST   /api/auth/login          # 로그인
POST   /api/auth/register       # 회원가입
POST   /api/auth/logout         # 로그아웃
POST   /api/auth/verify-phone   # 휴대폰 인증
```

#### Job Seeker
```
GET    /api/users/me                    # 내 정보 조회
PATCH  /api/users/me                    # 내 정보 수정
GET    /api/resumes                     # 이력서 목록
POST   /api/resumes                     # 이력서 생성
GET    /api/resumes/:id                 # 이력서 상세
PATCH  /api/resumes/:id                 # 이력서 수정
DELETE /api/resumes/:id                 # 이력서 삭제
GET    /api/resumes/:id/pdf             # 이력서 PDF 변환
```

#### Job Postings
```
GET    /api/jobs                        # 공고 목록
GET    /api/jobs/:id                    # 공고 상세
GET    /api/jobs/search                 # 공고 검색
GET    /api/jobs/recommended            # 추천 공고
```

#### Applications
```
GET    /api/applications                # 지원 내역
POST   /api/applications                # 지원하기
GET    /api/applications/:id            # 지원 상세
DELETE /api/applications/:id            # 지원 취소
```

#### Company
```
GET    /api/companies/me                # 내 기업 정보
PATCH  /api/companies/me                # 기업 정보 수정
POST   /api/companies/postings          # 공고 등록
GET    /api/companies/postings          # 내 공고 목록
PATCH  /api/companies/postings/:id      # 공고 수정
DELETE /api/companies/postings/:id      # 공고 삭제
GET    /api/companies/applicants        # 지원자 목록
PATCH  /api/companies/applicants/:id    # 지원자 상태 변경
```

## 9. 보안 요구사항

### 9.1 인증 및 권한
- Supabase Auth를 통한 JWT 기반 인증
- Row Level Security (RLS) 정책으로 데이터 접근 제어
- 역할 기반 접근 제어 (RBAC): job_seeker, company, admin

### 9.2 데이터 보호
- HTTPS 필수
- 민감 정보 암호화 저장
- 개인정보 마스킹 처리

### 9.3 입력 검증
- Zod를 통한 서버/클라이언트 양측 유효성 검증
- SQL Injection 방지 (Supabase 클라이언트 사용)
- XSS 방지 (React의 기본 이스케이프)
